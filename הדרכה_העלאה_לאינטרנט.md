## שלב 1: הכנת הפרויקט להעלאה

### 1.1 בניית האפליקציה (Build)
בטרמינל, מתוך תיקיית הפרויקט:
```powershell
cd "c:\Users\USER\Desktop\my_app"
npm run build
```
אחרי שזה מסתיים, נוצרת תיקייה בשם `dist` עם כל הקבצים המוכנים.

### 1.2 בדיקה שהבנייה הצליחה
וודא שיש תיקייה `dist` ובפנים קבצים כמו `index.html` ו-`assets`.

---

## שלב 2: העלאת האפליקציה ל-Vercel (חינם)

Vercel נותן אירוח חינם לאתרי React/Vite ומאפשר גישה 24/7.

### 2.1 פתיחת חשבון
1. היכנס לאתר: **https://vercel.com**
2. לחץ **Sign Up**
3. הירשם עם **GitHub** או **אימייל**

### 2.2 חיבור הפרויקט (אם יש לך GitHub)
1. העלה את הפרויקט ל-GitHub (אם עדיין לא):
   - צור Repo חדש ב-GitHub
   - העלה את תיקיית `my_app` (בלי `node_modules`!)
2. ב-Vercel: **Add New** → **Project**
3. בחר את ה-Repo של הפרויקט
4. **Framework Preset:** Vite (Vercel יזהה אוטומטית)
5. **Root Directory:** השאר ריק
6. **Build Command:** `npm run build`
7. **Output Directory:** `dist`
8. לחץ **Deploy**

### 2.3 חיבור בלי GitHub (עם Vercel CLI)
1. התקן Vercel CLI:
   ```powershell
   npm install -g vercel
   ```
2. מתוך תיקיית הפרויקט:
   ```powershell
   cd "c:\Users\USER\Desktop\my_app"
   vercel
   ```
3. ענה על השאלות (Login, שם הפרויקט, וכו')
4. בסוף תקבל קישור כמו: `https://my-app-xxx.vercel.app`

### 2.4 משתני סביבה (Environment Variables)
אם אתה משתמש ב-Supabase, הוסף ב-Vercel:
1. **Project** → **Settings** → **Environment Variables**
2. הוסף:
   - `VITE_SUPABASE_URL` = ה-URL של Supabase
   - `VITE_SUPABASE_ANON_KEY` = המפתח הציבורי

ואז עשה **Redeploy** לפרויקט.

## שלב 3: שרת המיילים 24/7

האפליקציה שולחת מיילים דרך שרת Node (`server/index.js`). כדי שזה יעבוד 24/7 יש שתי דרכים:

### אופציה א: שירות מייל חיצוני (מומלץ – בלי שרת משלך)
במקום להריץ שרת Node אצלך, אפשר לשלוח מיילים דרך שירות כמו:
- **Resend** (חינם עד כמות מסוימת)
- **SendGrid** (חינם עד 100 מיילים ליום)
- **Mailgun**

זה דורש שינוי קטן בקוד: במקום לשלוח ל-`localhost:4000`, הקריאה תלך ל-API של השירות. אם תרצה, אפשר לעבור על זה צעד-אחר-צעד בקוד.

### אופציה ב: הפעלת השרת על מחשב/שרת שפועל 24/7
- **מחשב ביתי** – להשאיר מחשב דלוק ולהריץ:
  ```powershell
  cd "c:\Users\USER\Desktop\my_app\server"
  node index.js
  ```
  (פחות מומלץ: חשמל, כיבויים, וצריך פתיחת פורט ברשת).
- **שרת VPS** (למשל DigitalOcean, Linode, או אפילו Oracle Free Tier):
  - מעלים את תיקיית `server` לשרת
  - מריצים שם `node index.js` עם **pm2** או **systemd** כדי שהתהליך ירוץ כל הזמן ויעלה אחרי הפעלה מחדש.

## שאלות נפוצות

**ש: האם זה באמת חינם?**  
ת: Vercel ו-Supabase יש תוכניות חינם. שירותי מייל יש חינם עד כמות מסוימת.

**ש: הנתונים שלי ב-localStorage – מה איתם?**  
ת: אם תעבוד עם Supabase (כמו שהכנו), הנתונים יהיו במסד הנתונים בענן. אם תרצה להעביר מה-localStorage ל-Supabase, יש פונקציית מיגרציה בפרויקט.

**ש: איך מתקנים משהו אחרי שהעליתי?**  
ת: מעדכן קוד, עושה שוב `git push` (אם חיברת GitHub) או `vercel --prod`, ו-Vercel יעלה גרסה חדשה אוטומטית.
