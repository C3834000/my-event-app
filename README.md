# ME-CFM – ניהול אירועים (קליכיף)

אפליקציית ניהול אירועים, לקוחות, לידים ומשימות. כולל טופס הזמנה ציבורי, פורטל לקוח ושליחת מיילים.

---

## איך להריץ את האפליקציה

### דרישות
- **Node.js** (גרסה 18 או חדשה יותר)

### 1. התקנת תלויות
```bash
npm install
```

### 2. הרצת הממשק (פרונט)
```bash
npm run dev
```
האפליקציה תיפתח ב־**http://localhost:3000**.

### 3. (אופציונלי) הרצת שרת המיילים
כדי ששליחת מיילים תעבוד (אישור הזמנה, קישור לפורטל וכו') צריך להריץ את השרת במקביל:

```bash
# טרמינל ראשון – הפרונט
npm run dev

# טרמינל שני – שרת המיילים
npm run dev:server
```

או להריץ את השרת ידנית:
```bash
cd server
npm install
node index.js
```
השרת רץ על **http://localhost:4000**. ה־Vite מפנה קריאות מ־`/api` לשרת הזה.

## שמירת נתונים

- **איפה נשמר:** בדפדפן ב־**localStorage** (מפתח: `ME_CFM_STORAGE_V12`).
- **מה נשמר:** אירועים, לקוחות, לידים, משימות, הגדרות, וטפסים מותאמים.
- **מגבלה:** הנתונים מקומיים למחשב ולדפדפן. מחיקת נתוני האתר / דפדפן אחר = אובדן הנתונים. לעתיד אפשר להוסיף מסד נתונים בשרת.

---

## שליחת מיילים

### הגדרה
1. העתק את הקובץ `server/.env.example` ל־`server/.env`.
2. מלא ב־`server/.env` את פרטי ה־SMTP (שרת המייל שלך).

**דוגמה – Gmail:**
- ב־Gmail: הפעל "אימות דו־שלבי" ויצירת "סיסמת אפליקציה".
- ב־`server/.env`:
  ```
  SMTP_HOST=smtp.gmail.com
  SMTP_PORT=587
  SMTP_USER=your-email@gmail.com
  SMTP_PASS=your-app-password
  FROM_EMAIL=your-email@gmail.com
  FROM_NAME=קליכיף
  ```

**ספק אחר (Outlook, etc.):**  
השתמש ב־host ו־port של הספק והכנס משתמש וסיסמה.

### מתי נשלחים מיילים
- **אחרי שליחת טופס הזמנה** (`/book`) – מייל אישור ללקוח.
- **"שלח פורטל במייל"** בלוח לידים – מייל עם קישור לפורטל הלקוח.
- **"שלח טופס הזמנה במייל"** (אם קיים בממשק) – מייל עם קישור לטופס ההזמנה.

אם השרת לא רץ או ש־SMTP לא מוגדר, הפעולות ימשיכו לעבוד אבל המייל לא יישלח (ותופיע התנהגות/הודעה בהתאם).

## בנייה לפריסה
```bash
npm run build
```
הקבצים ייווצרו בתיקייה `dist`. אפשר להעלות אותם לשרת סטטי (Netlify, Vercel, וכו').

**חשוב:** בפריסה אמיתית צריך להריץ גם את שרת המיילים (או להעביר את ה־API לשרת/פונקציה בענן) ולוודא שה־URL של האתר מתאים (למשל לשליחת קישורים במייל).

## מבנה תיקיות (בקצרה)
- `components/` – קומפוננטות משותפות (Layout, Sidebar).
- `context/` – Context גלובלי (נתונים ופעולות).
- `pages/` – דפי האפליקציה (לוח אירועים, לקוחות, לידים, משימות, הגדרות, טופס הזמנה, פורטל).
- `services/` – לוגיקה חיצונית (mockData, utils, emailService).
- `server/` – שרת Express לשליחת מיילים (API: POST `/api/send-email`).
